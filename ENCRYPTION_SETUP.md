# Настройка шифрования базы данных

## Проблема с pysqlcipher3 на Windows

`pysqlcipher3` требует компилятор Microsoft Visual C++ 14.0 или выше для установки на Windows. Это может быть проблематично.

## Решение: Использование cryptography (рекомендуется)

Бот поддерживает альтернативный метод шифрования данных на уровне приложения с помощью библиотеки `cryptography`, которая устанавливается без компилятора.

### Установка

```bash
pip install cryptography
```

### Настройка

1. Добавьте ключ шифрования в файл `.env`:
   ```
   DB_ENCRYPTION_KEY=your_secret_key_here_min_32_chars_recommended
   ```

2. Перезапустите бота.

### Как это работает

- **Шифрование данных**: Чувствительные поля (ситуация, мысли, доказательства, заметки) шифруются перед сохранением в БД
- **Автоматическое расшифрование**: Данные автоматически расшифровываются при чтении
- **Прозрачность**: Для остального кода шифрование прозрачно

### Альтернатива: SQLCipher (требует компилятор)

Если вы хотите использовать полное шифрование БД на уровне файла:

1. Установите Microsoft C++ Build Tools: https://visualstudio.microsoft.com/visual-cpp-build-tools/
2. Установите SQLCipher:
   ```bash
   pip install pysqlcipher3
   ```
3. Добавьте ключ в `.env` как указано выше

### Рекомендация

Для большинства случаев использования **рекомендуется использовать `cryptography`**, так как:
- ✅ Легко устанавливается на Windows
- ✅ Не требует компилятора
- ✅ Шифрует все чувствительные данные
- ✅ Достаточно безопасно для большинства применений
