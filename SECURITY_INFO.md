# Информация о безопасности базы данных

## Текущее состояние

**База данных НЕ зашифрована.** Используется стандартный SQLite без шифрования.

## Рекомендации по безопасности

### Вариант 1: SQLCipher (рекомендуется)
SQLCipher - это расширение SQLite с полным шифрованием базы данных.

**Установка:**
```bash
pip install pysqlcipher3
```

**Использование:**
```python
from pysqlcipher3 import dbapi2 as sqlite3

# Подключение с паролем
conn = sqlite3.connect('cognitive_diary.db')
conn.execute("PRAGMA key='your-secret-key'")
```

### Вариант 2: Шифрование на уровне файловой системы
- Использовать зашифрованные диски/разделы
- Использовать BitLocker (Windows) или LUKS (Linux)

### Вариант 3: Шифрование чувствительных полей
Шифровать только чувствительные данные перед сохранением в БД.

## Текущие меры безопасности

1. ✅ Параметризованные запросы (защита от SQL-инъекций)
2. ✅ Валидация всех входных данных
3. ✅ Rate limiting
4. ✅ Мониторинг подозрительной активности
5. ❌ Шифрование базы данных (не реализовано)

## Важно

База данных содержит персональные данные пользователей (записи дневника мыслей).
Рекомендуется добавить шифрование для соответствия требованиям GDPR и защиты данных.
